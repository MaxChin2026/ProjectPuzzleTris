# 07 — 版本规划

## 总体目标

> [!abstract] MVP 核心命题
> **验证基础玩法的合理性**：确跭「5选1方块机制（软约束）+ 消行→子弹攻击 + 技能格塔防 + 波次式敌人压力」四者结合后，能否产生健康的心流节奏和战斗爽感。
>
> **不做的事**：不发散至多英雄养成、RPG成长、社交系统等外围系统，聚焦核心玩法闭环。

---

## 版本阶段规划

### v0.1 — 棋盘原型（框架验证）

**目标**：验聦5选1方块机制的基础可玩性

| 功能点 | 优先级 | 状态 |
| ------ | ------ | ---- |
| 10×20 棋盘渲染 | P0 | ⬜ 待开发 |
| 7种方块形状 + 全旋转状态 | P0 | ⬜ 待开发 |
| 5退1插槽系统 | P0 | ⬜ 待开发 |
| 点触/拖拽/释放交互 | P0 | ⬜ 待开发 |
| Ghost Piece 落点预判 | P0 | ⬜ 待开发 |
| 快速下落动画（非瞬移） | P0 | ⬜ 待开发 |
| 消行检测与消除动画 | P0 | ⬜ 待开发 |
| 边界限制（水平移动） | P0 | ⬜ 待开发 |
| **软约束生成逻辑** | P1 | ⬜ 待开发 |

**验证问题**：
- [ ] 5选1机制是否有足够的决策趣味？
- [ ] 无旋转操作是否会让玩家感到被动/受限？
- [ ] 软约束是否有效避免「全废牌」而不破坏随机感？
