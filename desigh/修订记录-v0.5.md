# 修订记录 — v0.5

## 变更概要

| 类别 | 变更内容 |
| ---- | -------- |
| 难度调整 | 方块选择从 **3选1** 改为 **5选1** |
| 去重约束 | 同一轮5个候选槽中不生成相同形状（旋转不同也视为重复） |
| 旋转平衡 | 根据现有槽横/纵方块数量，偏向生成朝向较少的旋转方向 |
| 波次出怪 | 敌人改为波次式出现（5只起，密集出现→波间冷却），替代原4秒单只间隔 |
| 取消机制 | 拖拽后可拖回槽区放手取消，或直接点击其他槽位换选 |
| ⚠️ 机制移除 | **移除底部增行压力机制**，棋盘压力改由敌人漏怪承担 |
| 代码修改 | `SLOT_COUNT` 5槽、形状去重、旋转平衡、波次系统、取消机制；移除`ADD_ROW_INTERVAL_MS` |
| 文档同步 | 全量更新涉及「3选1」的设计文档；新增波次节奏；标记底部增行为已移除 |

## 详细变更

### 代码变更

| 文件 | 变更内容 |
| ---- | -------- |
| `code/src/constants.ts` | `SLOT_COUNT = 3` → `SLOT_COUNT = 5` |
| `code/src/BlockManager.ts` | 注释更新：「3-slot」→「5-slot」；`refillSlots` 新增已有形状集合传递；`_generateSingle(excludeShapes)` 新增形状去重逻辑（不同旋转也视为同形状） |
| `code/src/Renderer.ts` | 槽内预览格子尺寸 `cellSz: 14` → `12`（适配5槽宽度，避免I形方块预览溢出） |

#### 形状去重逻辑说明

```
refillSlots() 生成第 i 槽时：
  1. 收集其余 4 槽已有形状 → excludeShapes
  2. _generateSingle(excludeShapes) 最多尝试 10 次：
     - 随机生成候选 → 若 shape ∈ excludeShapes，跳过
     - 若 canPlaceAnywhere → 返回
  3. 放宽fallback：遍历未被排除的形状中找可放置者
  4. 最终fallback：I形横向（棋盘接近满 → GAME OVER）
```

> 7种形状 / 5个槽 → 理论上始终可以保证5槽形状各异（极端情况下保底）。

> [!note] 布局计算说明
> 棋盘总宽 = 10 × 28 = 280px。5个槽 + 间距（gap=4）：`(280 - 6×4) / 5 ≈ 51px` 每槽。
> I形方块4格横向预览 = 4 × 12 = 48px < 51px，可正常容纳。

## AI 设计评审意见

### 本次修改评估

此次 3选1→5选1 的改动是一次直接的难度下调，但需要仔细权衡其对游戏设计整体的影响。

---

### 批判性分析

#### ✅ 改动的合理性

- **「无牌可用」问题是核心体验破坏点**：当玩家面对 3 个都无法合理放置的方块时，游戏变成了等待失败的无力感，而非策略挑战。5选1 增加可用方块数量，在不改变棋盘压力的前提下提供更多决策出路。
- **决策空间扩大但不发散**：5个候选中选1个，仍保留「选哪块」作为核心决策，且选项更多意味着组合策略丰富（如「为消行储备合适形状」），决策深度不降反升。
- **与软约束协同**：原有软约束旨在避免全废牌，5选1后极端废牌局面概率大幅下降，软约束触发频率降低，随机感更自然。

#### ⚠️ 潜在风险

1. **压力曲线变宽松**：5选1让玩家几乎总能找到合适方块，棋盘压力来源变得单一（仅剩底部增行和敌人漏怪）。如果后期没有对应加压设计，游戏节奏可能趋于平淡。
   - **建议**：加速底部增行间隔或增加中后期阶段敌人波次密度，弥补难度空缺。
   
2. **UI 空间紧张**：5个槽在当前 10 列 × 28px = 280px 宽度下每槽约 51px，I形方块预览刚好能容纳（cellSz=12），但视觉上会比较拥挤，移动端触摸精度可能成为问题。
   - **建议**：考虑对方块槽区域高度做加高（SLOT_AREA_H 从 80 增至 90），或在移动端使用更大的触摸热区。
   
3. **技能格密度增加**：5选1意味着每轮可见的技能格候选增多，技能格获取频率上升，可能过度削弱「技能格是稀有奖励」的心理价值。
   - **建议**：考虑将 `SKILL_INJECT_CHANCE`（当前 0.30）从每块独立概率调整为「5块中最多2块含技能格」的上限约束，维持技能格稀缺感。

---

### 改进建议（优先级排序）

| 优先级 | 建议                                                   | 预期效果          |
| --- | ---------------------------------------------------- | ------------- |
| P0  | **监控游戏时长**：移除底部增行后无棋盘内压，玩家可能长时间存活但缺乏紧迫感，需通过波次难度加速来弥补 | 保持适度紧张感       |
| P1  | **监测中后期底部增行间隔**～**敌人波次调参**：若玩家平均存活时间过长则加快波次节奏        | 弥补5选1带来的压力下降  |
| P2  | **技能格上限约束**：5块中至多2块含技能格                              | 维持技能格稀缺感与奖励价值 |
| P3  | **移动端触摸热区优化**：5槽触摸区域增大，防止误触                          | 改善操作手感        |
| P4  | **考虑重新引入底部增行但以更温和的形式**：例如仅在高波次阶段触发，或作为Boss专属机制       | 高难度下恢复棋盘内压    |
