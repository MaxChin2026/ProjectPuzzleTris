# 修订记录 — v0.2（2026-04-08）

> [!abstract] 本次修订摘要
> 根据[[设计概念修改]]，对原[[初始设计概念]]进行重大架构级变更：**完全移除RPG/战场系统**，将上半屏幕的"横向战场 + 英雄弓箭手"替换为「干扰方块预警区」；奖励格由生命回复/主动技能改为强力棋盘消除效果。

---

## 变更清单

### ❌ 移除的设计

| 移除模块 | 原设计内容 | 移除原因 |
| -------- | ---------- | -------- |
| 横向战场区域 | 屏幕上半部显示弓箭手英雄与从右向左移动的敌人 | 与俄罗斯方块玩法割裂，两块内容各自独立运行，无机械联动 |
| 英雄系统 | 英雄HP、主动技能（穿透箭/燃烧箭/冰箭）、被动技能树（攻速/暴击等）| 战场移除后英雄系统失去载体 |
| RPG成长机制 | 经验值 → 升级 → 3选1被动技能 | 成长系统依附于英雄战斗，随战场一并移除 |
| 敌人系统 | 敌人从右方召唤、向左移动、撞击造成伤害、精英敌人触发多缺口行 | 失去战场后无意义 |
| 满棋盘阶梯扣血 | 满棋盘时按当前最大HP百分比阶梯式扣除（5%/10%/15%/20%）| RPG生命值体系移除；即时GAME OVER逻辑更清晰 |
| 奖励格：生命回复 | 奖励格触发时回复英雄HP | 移除HP系统后无意义 |
| 奖励格：主动技能释放 | 奖励格消除时释放英雄主动技能（穿透箭等）| 移除技能系统后无意义 |

---

### ✅ 新增的设计

| 新增模块 | 设计内容 | 作用 |
| -------- | -------- | ---- |
| 干扰方块预警系统 | 棋盘顶部设置预警区，显示即将从顶部投放的干扰方块；预警期 2~3 秒，给玩家有限应对窗口 | 替代RPG压力，形成「顶部突发压力」对抗底部持续增行压力 |
| 干扰方块形状 | 专属小型形状（单格/横条2格/L形3格），非标准Tetris七种，视觉上与玩家方块明确区分 | 增加棋盘混乱度，且难以通过单次消行清除 |
| 奖励格：行消除 | 消行触发时，强制消除棋盘最靠近底部的不完整行 | 直接化解底部堆高压力 |
| 奖励格：列消除 | 消行触发时，清空奖励格所在列所有方块 | 化解竖向死角堆叠 |
| 奖励格：填空缺 | 消行触发时，将最底层不完整行所有空格填满（连锁触发消行）| 加速接近完整的行消除 |

## AI 批判性分析与改进建议

> [!warning] 以下内容为AI对当前v0.2设计的批判性审查，目的是识别潜在设计风险并提出改进建议，供设计决策参考。

---

### 问题一：「无旋转」设计的被动感风险

**现状描述**
3选1方块槽移除了玩家主动旋转操作——3个方块的形状和旋转在生成时已固定，玩家唯一的空间操作是选哪块方块、方块落在哪一列。

**批判性分析**
这个设计存在两种极端体验的风险：
- **幸运时**：三个方块中恰好有一个完美契合当前棋盘缺口，决策清晰，满足感强；
- **不幸时**：三个方块均为烂牌（如都是S/Z的不利旋转），玩家无法改变局面，只能被动承受，产生「无能感」而非「策略感」；
- 在顶部干扰方块频繁落下、底部增行加速的高压阶段，「三个方块都是烂牌」概率虽低但足以在关键时刻直接导致失败，玩家会归因于「运气太差」而非「策略失误」，削弱重复游玩动力。

**改进建议**
- 方案A（低成本）：引入「换牌」机制 —— 玩家每局可使用1~2次「洗牌」，重新随机生成3个方块。视洗牌时机为策略资源。
- 方案B（系统性）：设计「旋转代价」机制 —— 可旋转，但旋转会消耗少量时间或分数，让「使用随机旋转」和「花代价调整旋转」形成策略权衡。
- 方案C（分析期生成优化）：生成方块时对形状池做软约束，如限制同一旋转状态同时出现3次、避免三块全为S/Z/双斜块。降低「无解局面」出现概率，不影响设计意图。

**推荐优先验证**：方案C（实现成本最低，不破坏现有交互设计），方案A次之。

### 问题二：干扰方块与主动放置方块的空间冲突缺乏协同感

**现状描述**
干扰方块从顶部预警落下，落入某列栈顶位置；同时玩家正在处理底部增行的压力，选择并放置3选1方块。两个机制在同一棋盘运行，但在决策层面目前是**各自独立**的——玩家放置预测不影响干扰列的选取，干扰方块也不主动「配合」玩家的薄弱区域。

**批判性分析**
- 当前干扰列为「纯随机选取」，有时干扰方块落入已经很高的堆叠区域（雪上加霜），有时落入平整区域（影响极小）；随机性使压力体验方差过大，高压时不公平，低压时形同虚设；
- 玩家无法通过「提前铺垫」干扰列来战略性消化干扰方块，只能在预警期2~3秒内被动响应，决策深度受限；
- 干扰方块→落地→被后续玩家方块消行消除的「化解循环」体验链尚不明确，玩家不能直观预判「消掉干扰需要几步」。

**改进建议**
- 建议A（权重干扰列选择）：干扰列倾向于选取当前棋盘**高度处于中等位置**的列（而非最高或最低），确保干扰落地后能形成「有挑战但可解决」的棋盘形态，而非直接推满；
- 建议B（干扰方块视觉强化）：干扰方块落地后视觉染色持续高亮（与普通方块区别），让玩家能清晰追踪干扰方块的位置，以便规划消行优先级；
- 建议C（预警区信息增强）：预警区除显示干扰形状外，同时在棋盘该列的侧边显示「干扰后该列预计高度」，使玩家能在预警期内做出更有依据的应对决策。

**推荐优先验证**：建议A（平衡性直接可测），建议B（工程成本低且提升关键信息可读性）。

### 问题三：三种奖励格效果的价值感和选择张力待验证

**现状描述**
行消除/列消除/填空缺三种奖励格效果均指向「消除」这一单一维度，且触发逻辑均依赖「含奖励格的行被消除」这一前置条件。

**批判性分析**
- **效果同质感**：三种奖励本质上都是「删除某些格子」，仅作用范围不同（水平行 / 垂直列 / 填满行），可能让玩家感受不到明显的类型差异；
- **自我参照悖论**：奖励格需要先被消行触发，而行消除奖励的作用也是消行——这意味着当玩家最需要「行消除」奖励时（棋盘很高），往往已经难以构造消行条件来触发它，奖励的实际效用与棋盘最高压的时刻错位；
- **填空缺**的效果是填满不完整行来触发消行，本质上是「帮玩家完成差一格就能消除的行」，但若棋盘最底层行只有少数格子，填空缺实际上会「浪费」格子（填了但不消），反而增加棋盘高度；
- 三种奖励中，**列消除**在语义上最强（清空整列），但因为按格计分（低于整行消行分），与其强效果相比激励感较弱。

**改进建议**
- 建议A（效果差异化）：考虑增加非主动消除类奖励，如「方块变形」（将当前选中方块临时变换为1×1单格，方便填缝）或「冻结增行」（暂停底部增行倒计时3轮），打破奖励类型的同质性；
- 建议B（触发条件松绑）：允许玩家「长按」奖励格所在方块来提前激活奖励（无需等待消行），但需要消耗一定分数作为代价，降低「奖励拿不到就过期」的挫败感；
- 建议C（填空缺逻辑优化）：填空缺目标行选取改为「缺口最少的不完整行」（而非固定取最底层），提高其实际触发消行的概率，使效果更可靠。

**推荐优先验证**：建议C成本最低且直接改善填空缺的体验一致性，建议A作为中期迭代方向。

### 问题四：满棋盘即时GAME OVER与双重压力的容错边界

**现状描述**
本次修订将满棋盘惩罚从「阶梯扣血」改为「即时GAME OVER」，核心理由是逻辑更清晰。

**批判性分析**
- 双向夹击压力（底部增行 + 顶部干扰方块）在节奏上可能出现两种机制同时达到峰值的「共振时刻」（如底部增行即将触发，同时有干扰方块落下），此刻棋盘压力急剧升高，即时GAME OVER门槛在高难阶段可能导致玩家感到不公平（「我还在应对干扰，底部就又加行了」）；
- 相比ROG脱出型游戏，纯消除游戏的「一局」时间可能较长，即时GAME OVER在中后期会让玩家损失大量游玩时间，重复游玩挫败感较强；
- 原阶梯扣血虽逻辑复杂，但提供了「危险警告」功能——第一次满棋盘给玩家一个信号，让玩家知道已经到极限；即时GAME OVER将这个警告信号消除，玩家对「还剩多少空间」的感知依赖棋盘视觉，可能不够直观。

**改进建议**
- 建议A（缓冲机制）：保持即时GAME OVER，但在方块无法正常生成时给予**1~2秒的视觉警告**（棋盘边框闪红、强烈音效），让玩家有「最后反应机会」的感知（即使实际上结果已确定）；
- 建议B（危险区可视化）：在棋盘顶部3行区域使用渐变色（正常色→橙→红）实时标示「危险高度」，让玩家在达到满棋盘之前就能清晰感知风险；
- 建议C（软边界探索）：对"满棋盘"的定义作更精细的处理——当新方块生成时，若顶部空间不足，先尝试将方块生成位置上移2格（视觉溢出区），若此时玩家能立刻消行（类似Tetris的"顶出"消行），则不判定GAME OVER，给予最后一次化险为夷的机会。

**推荐优先验证**：建议A和B成本极低，应在v0.2版本中作为基础体验项直接实现。
