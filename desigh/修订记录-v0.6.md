## 详细变更

### Bug 修复：子弹优先攻击最接近终点的敌人

**问题描述**

`BulletSystem._selectTarget` 中存在"越杀跳目"逻辑：若最近敌人的 HP ≤ 行伤害 × `OVERSKILL_THRESHOLD`（0.5），则跳过该敌人，将火力转向血量更高的次近敌人。

这导致在常规战斗中，HP 较低的最前线敌人被忽略，火力被浪费在更远的敌人上，最终使最接近出口的敌人成功漏怪。

**修复方案**

移除越杀跳目判定，仅保留"跳过本批次伤害已预测死亡的敌人"的逻辑。现在 `_selectTarget` 始终返回 `getSortedByProgress()` 中第一个存活的敌人（即最接近终点者）。

```typescript
// 修复前（错误逻辑）
if (hp <= rowTotalDamage * OVERSKILL_THRESHOLD) {
  // 跳转到次近敌人 → 导致最近敌人逃脱
  return nextEnemy ?? enemy;
}

// 修复后（正确逻辑）
// 只跳过本批次预测已死亡的敌人，其余一律优先打最近的
if (hp <= 0) continue;
return enemy;
```

### 新功能：横向拖动卡入空隙（挤入机制）

**功能描述**

修改 `BlockManager.moveActive`：当横向移动（`dCol !== 0, dRow === 0`）在当前行发生碰撞时，自动向下逐行寻找第一个可放置的行，并将方块移至该位置。

**行为逻辑**

```typescript
// 横向移动遇到障碍时，向下滑动寻找第一个有效行
if (dRow === 0 && dCol !== 0) {
  for (let r = this.active.row + 1; r < ROWS; r++) {
    if (this.canPlace(this.active.def, r, newCol)) {
      this.active.row = r;   // 方块下沉到有效位置
      this.active.col = newCol;
      return true;
    }
  }
}
```

**设计原则**

- 方块只能因横向移动而**下沉**，不会上浮（符合重力逻辑）
- 每次横向移动一格时，自动寻找目标列中当前行及以下的第一个可落点
- 与 `hardDrop` 兼容：卡入后继续 hardDrop 仍会将方块落至最低有效位置

## AI 设计评审与改进建议

### 当前玩法批判性分析

#### 1. 越杀跳目机制的设计初衷与实际效果

原 `OVERSKILL_THRESHOLD` 机制的初衷是减少伤害浪费（避免对 1 HP 敌人打出 100 伤害的越杀）。在 MOBA/RPG 背景下这是合理的资源优化，但在塔防场景中方向完全相反：

- **塔防核心威胁**：敌人到达终点（漏怪）才是失败风险，而非伤害效率。
- **越杀不是代价**：子弹数量由消行驱动，不存在"资源"损耗，越杀仅是视觉上的浪费。
- **跳目的实际危害**：跳过最近敌人→最近敌人继续前进→在下次消行前可能漏怪。

**结论**：该机制与塔防目标南辕北辙，正确做法是**无条件优先最近敌人**。

#### 2. 当前预测 HP 机制的价值

保留同批次多行消除的预测 HP 机制（`predictedHp` Map）是合理的：若某行的子弹已经能杀死最近敌人，下一行应自动转向第二近的敌人，避免所有行都打向同一个即将死亡的目标。这个机制现在运作正确。

#### 3. 挤入机制的潜在问题

**问题**：若玩家拖动方块经过一个浅层空隙（如仅 1 格深），方块会"卡住"在该位置，无法继续向右移动（因为后续格也在同一行有障碍）。玩家可能误以为方块卡住了而无法操作。

**建议**：在 UI 上对方块的当前位置显示幽灵预览（ghost piece），使玩家能直观看到方块在目标列的落点，减少误操作。（当前已有 `getGhostRow` 用于此目的，确认 Renderer 已调用即可。）

#### 4. 改进建议

**建议 A：在飞行中的子弹也纳入预测伤害计算**

当前 `predictedHp` 只考虑"本次消行产生的子弹"，不考虑已在飞行中的子弹（上次消行产生、尚未到达的子弹）。若多次快速消行，后一批子弹可能重复锁定已被前一批子弹秒杀的目标。

- **改进方案**：在 `fireBulletsForRows` 初始化 `predictedHp` 时，遍历 `this.bullets`（飞行中子弹），将其 `damage` 从对应目标的 HP 中预扣。
- **预期效果**：减少因子弹传播延迟导致的目标浪费；多路消行时火力自动分散到多个敌人。

**建议 B：对 Boss/精英敌人增加"临界威胁"提示**

当最近敌人的 `pathProgress > 0.85`（距终点 <15%）时，在 UI 上显示红色警告边框，提示玩家需要立即消行。这能将现有的"最近敌人优先"机制转化为可感知的游戏压力反馈。

**建议 C：允许技能"强制目标锁定"覆盖默认排序**

未来可引入"锁定目标"技能，允许玩家手动指定高威胁敌人（如血量虽低但速度极快的敌人）为优先目标，打破默认的 pathProgress 排序。此机制能增加策略深度，同时尊重默认的"最近优先"原则。
